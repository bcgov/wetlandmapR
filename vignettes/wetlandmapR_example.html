wetlandmapR Example

Deepa Filatow, Gillian Harvey, Alexandre Bevington, Hunter Gleason

Using example data provided with the package the following vignette demonstrates the use of the wetlandmapR package to classify wetlands using terrain derivatives and external inputs such as satellite imagery. This document is written to serve as a reference for performing similar classification in any area of interest.

Using the wetlandmapr Docker …

A Docker image based on the rocker/geospatial:3.6.3 image has been assembled for running the wetlandmapR package package. Please see the Docker Install page for platform specific guidance on installing Docker. Assuming a local installation of Docker is in place, the huntgdok/wetlandmapr:latest Docker image can be downloaded by passing the command below from your terminal or command prompt:

docker pull huntgdok/wetlandmapr:latest

once the image has been successfully pulled, it can be run from the terminal or command prompt using the docker run call as shown below

docker run -e PASSWORD=APassword -e ROOT=TRUE -p 8787:8787 -v "/c/User/Desktop/Data:/home/rstudio/Data" --rm huntgdok/wetlandmapr:latest

where APassword is any password you want to use to log into rocker. To use with sudo privileges, add -e ROOT=TRUE to the docker run call as shown above. The -v flag in the Docker run call above mounts a volume on the host machine, in this case /c/User/Desktop/Data on a Windows machine, to a directory called Data on the Docker container, see Use Volumes for more information about mounting volumes. To view the running docker, go to the local URL localhost http://localhost:8787 in your internet browser. When prompted for a username and password, supply rstudio as a username, and the password that you set with the docker run call. You should then have a R-Studio session running with all the R and third party dependencies required for running the wetlandmapR package.

Alternatively, the package can be run locally assuming all of the required dependencies are installed.

Creating elevation derivatives …

The following lines of code define an output directory for the terrain derivatives, load in an example digital elevation model (DEM) from the wetlandmapR package, and calls the wetlandmapR::create_dem_products function to generate terrain derivatives using SAGA-GIS and RSAGA. These terrain derivatives will be used to predict the wetland classes. As a pre-processing step, wetlandmapR::create_dem_products first hydrologically conditions the DEM by filling any sinks, a binary sinks layer is output under the name SINKS.sgrd. Optionally, if a stream vector is provided (stream_vec) it may be burned into the DEM by a distance (burn_val) in meters defined by the user. The terrain derivative layers specified by products (default all) are output to the directory provided to outdir. Using the param_vect and param_values parameters additional options can be provided to the SAGA modules, such as shown for the SAGA-GIS TPI module below.

#Establish a RSAGA environment 
env<-rsaga.env(parallel = T)

#Specify a output directory, we'll use the temporary directory for this example
out_dir<-paste(tempdir(),"/wetlandmapR_expl",sep="")
dir.create(out_dir)

#Load a example digital elevation model DEM from wetlandmapR package 
target_dem <- system.file('extdata','DEM.tif',package = 'wetlandmapR')

#Load freshwater atlas stream data from wetlandmapR package
streams <- system.file('extdata','streams.shp',package = 'wetlandmapR')

#Create SAGA elevation derivatives using the wetlandmapR 'create_dem_products' function, burn FWA streams into DEM by 2 m ...
create_dem_products(dem=target_dem,
                    stream_vec = streams,
                    burn_val=2,
                    outdir=out_dir,
                    param_vect = c("TPI"),
                    param_values = list(list(DW_WEIGHTING=2)),
                    env=env)

Creating the raster stack, adding external predictors …

In the code chunk below we create a list of paths to the raster images that we want to use to predict the different wetland classes, in this case most of the inputs were derived from the DEM using wetlandmapR::create_dem_products in the code above. However, in the example we add to this list a Sentinel-2 MSI Level-2A Band-4 (B4:Red) image as an additional predictor of the wetland classes. Note, any raster layer that the user wants to include for predicting the wetland classes can be added to the input list. We then call the wetlandmapR::stack_rasters function to create a SpatRaster object from the list of raster inputs specified by the rasters parameter. The outdir parameter is an optional output path defining where the stacked rasters are to be saved in ‘.tif’ format. When the aligned parameter is FALSE (default), the input raster will be re-sampled (bilinear) to the resolution and extent of a target raster provided by the target_raster parameter, in this case the provided DEM. The rastLUTfn is an optional parameter specifying the filename of a output raster lookup table (rastLUT) .csv file for use in wetlandmapR::wetland_map. The resulting SpatRaster object is plotted below. The rastNames parameter allows the user to specify the names of the output rasters, in this case just taken as the basenames without extensions.

#Get list of SAGA terrain derivatives within the output directory 
raster_list <- list.files(out_dir, "sdat$", full.names = TRUE)

#Drop the filled DEM and ASPECT layers from predictor list, as they are redundant. 
raster_list<-raster_list[!raster_list %in% 
                           c(file.path(out_dir,'ELEV.sdat'),
                             file.path(out_dir,'ASPECT.sdat'))]

#E.G., Add external Sentinel-2 Band 4 data to the list of raster inputs from wetlandmapR package, this could be any layer
raster_list<-append(raster_list,system.file("extdata", "B4_SENT2.tif", package = "wetlandmapR"))

#Stack all raster inputs using the wetlandmapR 'stack_raster' function, passing 'raster_list' as input, output a raster Look-Up-Table 'rastLUT' CSV 
raster_stack <- stack_rasters(rasters = raster_list,
                              target_raster = target_dem,
                              outdir = out_dir,
                              rastLUTfn = out_dir, 
                              rastNames = file_path_sans_ext(basename(raster_list)))

#Plot the new stack of raster objects (input layers)
plot(raster_stack)

Attribute raster values to training points …

This code chunck reads in example training points from the wetlandmapR package as a SpatVector object with the wetland classes coded by the T_W_Class_ field. In addition, biogeoclimatic zones (BEC) are loaded internally using the bcmaps package, model outputs will be grouped by this layer by setting it as the aoi parameter in the wetlandmapR::grid_values_at_sp function. Using the raster stack of predictors plotted above as the raster_stack parameter, the wetlandmapR::grid_values_at_sp function attributes the intersecting raster values to the training points provided as the points parameter. Optionally, the resulting table table can be written to the output directory (filename) as a CSV, in this case named “train_pnts_attributed.csv”.

#Load training data as SpatVector  
training_points<-vect(system.file('extdata', 'TrainingPnts.shp', package = 'wetlandmapR'))

#Clip training point to extent of the SpatRaster inputs, using sf for this
ext<-ext(raster_stack)
xmin = ext$xmin
xmax = ext$xmax
ymin=ext$ymin
ymax=ext$ymax
ext <- st_sfc(st_polygon(x = list(rbind(c(xmin, ymin), c(xmax, ymin), c(xmax, ymax), c(xmin, ymax), c(xmin, ymin)))), crs = st_crs(training_points))
training_points <- st_intersection(st_as_sf(training_points),ext)


#Load BEC zones from bcmaps 
aoi_polys<-bec(ask=F) %>% 
  st_crop(training_points)

#Plot AOIs and training points
ggplot()+geom_sf(data=aoi_polys,aes(fill=ZONE))+
  geom_sf(data=training_points)

Run model and diagnostics …

Below we fit a random forest wetland classification model by calling the wetlandmapR::wetland_model function using the training data that we attributed with the input raster layers above. We declare the response field to be the “T_W_Class” field, and set response.type as categorical. The “ID” field is specified for identifying unique observations. We assign the “BGC_ZONE” column to identify separate AOIs, and a separate classification model is fit for each unique AOI polygon. The responce.target parameter is for subsetting the training dataset, in this case we use all wetland categories. We also define the SINKS column (input) to be a factor variable. The results include a list containing model.build objects output from the ModelMap package for each AOI. Model outputs will be output to the model.folder directory, in this case “output”. Because the modeling process in random, a seed it set for reproducibility.

#Fit a random forest model 
model.out <- wetland_model(qdatafn = attributed_csv,
                           model.type = "RF",
                           model.folder = file.path(out_dir,"output"),
                           unique.rowname = "ID",
                           predList = predList,
                           predFactor = c('SINKS'),
                           response.name = "T_W_Class_",
                           response.type = "categorical",
                           seed = 42,
                           response.target = as.vector(unique(training_points$T_W_Class_)),
                           aoi.col = "BGC_ZONE")

Create map(s) from model …

Finally we call the wetlandmapR::wetland_map function which generates raster prediction surfaces using the model outputs from wetlandmapR::wetland_model, i.e., model.out for each mdoel / AOI. We point the function to the output directory assigned to wetland_model above. We also pass the rastLUT data frame, and BEC AOI SpatialPolygons (aoi), the AOI column (aoi.col) is defined as the ZONE column.

#Map random forest results 
wetland_map(model.out = model.out,
            model.folder = file.path(out_dir,"output"),
            rastLUTfn = rastLUT,
            aoi = aoi_polys,
            aoi.col = "ZONE")

Polygonize wetland classification map …

In this chunk we polygonize the resulting wetland classification map for the SBS BEC Zone generated by wetlandmapR::wetland_map using the wetlandmapR::raster_to_clean_polygon function. The classified raster for the SBS zone is provided as the input raster r, any polygons with an area less than 2500 square meters are removed (min_area_drop_m2), and any holes larger than 2500 square meters are filled (max_area_fill_m2). Optionally the resulting wetland polygons can be smoothed using the smoothr package by setting smooth as TRUE, and the desired smoothing method (smooth_method). The workers parameter defines the number of computer cores to use for the polygonization. The polygonized wetland classes are plotted below for the SBS model output.

Identify high accumulation pour points …

With the classified raster output converted to polygons, pour points for each polygon can be identified using a flow accumulation raster created in RSAGA. This is done by applying the wetlandmapR::get_low_high_loc function to each polygon (poly) and using the accumulation raster as input (lyr), and creating a matrix from the X-Y coordinates returned, along with a unique polygon identifier field UID. The low parameter return the location of the minimum raster value bounded by the provided polygon when TRUE, and maximum value when set to FALSE.

#Create a flow accumlation layer using RSAGA for target DEM
rsaga.geoprocessor(lib='ta_hydrology',
                          module=29,
                          param = list(DEM=target_dem,
                                       FLOW=file.path(out_dir,'ACC'),
                                       CONVERGENCE=1.1))


#Load the accumulation layer output from `set_grass_env` model as terra raster
acc<-rast(file.path(out_dir,'ACC.sdat'))

#Initialize an empty coordinates vector 
low_pnts <- c()
#Populate coordinated vector with coordinates of highest accumulation pixel within each wetland polygon 
for(i in 1:nrow(SBS_Class_ply))
{
  low_pnts<-rbind(low_pnts,cbind(get_low_high_loc(lyr=acc,poly=st_geometry(SBS_Class_ply[i,]),low=F),i))
}

Calculate upstream basin statistics …

In the code below the target DEM is hydrologicaly conditioned using the wetlandmapR::hydro_condition_dem function. Using the conditioned DEM upstream basins are delineated for the first 20 points in the low_pnts table, and zonal statistics computed for each of the input rasters in the raster_stack, listed in rastLUT, by applying the wetlandmapR::gen_upstream_basin and wetlandmapR::get_basin_stats functions. The result is a table of upstream basin statistics for each point, identified my a unique ID, which can be used for joining on.

#Hydrologic conditioning of target DEM
hcd<-hydro_condition_dem(dem=target_dem,
                         outdir=tempdir(),
                         env=env)

#Empty vector 
upstream_stats<-c()

#For the first 20 pour points in 'low_pnts' calculate upstream basin statistics
for(i in c(1:20))
{
  #Generate a binary grid representing the upstream area for the XY coordinates 
  gub<-gen_upstream_basin(dem=hcd,
                          x_coord = low_pnts$x[i],
                          y_coord = low_pnts$y[i],
                          outdir = tempdir(),
                          basin_id = i,
                          env=env)
  
  #Use the binary upstream grid as a zonal raster, calculate zonal statistics 
  basin_stats<-get_basin_stats(zone_rast=gub,
                               statlist= paste(rastLUT$V1,collapse = ';'),
                               outdir=tempdir(),
                               basin_id = i,
                               env=env)
  
  #Bind new statistics to the upstream_stats matrix 
  upstream_stats<-rbind(upstream_stats,basin_stats[2,])
  
  
}
